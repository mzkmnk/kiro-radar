# プロジェクト構造

## ディレクトリ構成

```
kiro-radar/
├── src/
│   └── main.rs          # メインアプリケーションロジック
├── target/              # ビルド成果物（Git 管理外）
├── .kiro/               # Kiro IDE 設定
│   └── steering/        # AI アシスタント用ガイドライン
├── Cargo.toml           # プロジェクト設定と依存関係
├── Cargo.lock           # 依存関係のロックファイル
├── README.md            # プロジェクト概要
└── LICENSE              # MIT ライセンス
```

## コード構成

### src/main.rs

現在の構造：

- **main()**: エントリーポイント、ターミナル初期化と復元
- **App 構造体**: アプリケーションの状態管理
  - `running`: アプリケーション実行状態
  - `new()`: インスタンス生成
  - `run()`: メインループ
  - `render()`: UI レンダリング
  - `handle_crossterm_events()`: イベント処理
  - `on_key_event()`: キーボード入力処理
  - `quit()`: アプリケーション終了

## アーキテクチャパターン

### イベント駆動アーキテクチャ

- crossterm によるイベント読み取り
- イベントに応じた状態更新
- 状態変更後の UI 再描画

### 単一責任の原則

- 各メソッドは明確な単一の責任を持つ
- イベント処理、レンダリング、状態管理を分離

## 今後の拡張予定

プロジェクトの成長に伴い、以下のような構造への拡張が想定される：

```
src/
├── main.rs              # エントリーポイント
├── app.rs               # App 構造体とメインロジック
├── ui/                  # UI コンポーネント
│   ├── mod.rs
│   ├── dashboard.rs     # ダッシュボード画面
│   └── widgets/         # カスタムウィジェット
├── spec/                # Spec ファイル解析
│   ├── mod.rs
│   ├── parser.rs        # Markdown パーサー
│   └── tracker.rs       # 進捗トラッキング
└── events.rs            # イベントハンドリング
```
